 Context

 The match continuity system (pixel-identical copy, reactive links, badges) is complete. Now we enhance the   
 reference continuity flow — used when adjacent shots have different camera angles but must share visual      
 continuity (same characters, environment, lighting, but different framing/composition).

 Current problems:
 1. "Pull/Push Ref" buttons silently add a URL to reference_image_order JSONB with no visual feedback beyond  
 a toast and tiny thumbnail
 2. No frame_links tracking for refs — can't show badges, can't resolve fresh URLs
 3. The camera_change generation path sends the reference image to Gemini with a metadata-only prompt — it    
 never analyzes what's actually in the reference image
 4. regenerateFrame() has no camera_change branch — falls through to generic generation
 5. Ref buttons styled as ghost (de-emphasized) while match buttons are outline

 User decisions:
 - Replace existing link when switching between match/ref (one link per frame, mutual exclusivity)
 - Show badges on BOTH source and target frames (emerald source, violet target)
 - After pulling ref, show badge + toast hint "regenerate to apply" (no auto-regen)
 - Always run vision analysis on every camera_change generation/regeneration (~$0.01-0.03 per call)

 // Find the target frame record
 const { data: targetFrame } = await supabase
     .from('frames')
     .select('id')
     .eq('shot_id', shotId)
     .eq('frame_type', targetFrameType)
     .single();

 if (targetFrame) {
     // Delete ANY existing link on target (match or ref — mutual exclusivity)
     await supabase.from('frame_links').delete().eq('target_frame_id', targetFrame.id);

     // Create reference link
     await supabase.from('frame_links').insert({
         source_frame_id: sourceFrameId,
         target_frame_id: targetFrame.id,
         link_type: 'reference',
     });
 }

 // Set start_continuity = 'camera_change' for start frames
 if (targetFrameType === 'start') {
     await supabase.from('shots')
         .update({ start_continuity: 'camera_change', updated_at: new Date().toISOString() })
         .eq('id', shotId);
 }

 5B. Keep reference_image_order in sync

 At generation time (inside generateWithCameraChangeAnalysis()), refresh the continuity URL in
 reference_image_order if the source frame's image has changed since the ref was pulled:

 // Resolve live source URL and update stale reference_image_order if needed
 const { data: currentRefOrder } = await supabase
     .from('shots').select('reference_image_order').eq('id', shotId).single();
 if (currentRefOrder?.reference_image_order) {
     const order = currentRefOrder.reference_image_order as any[];
     const entry = order.find((e: any) => e.type === 'continuity');
     if (entry && entry.url !== referenceImageUrl) {
         entry.url = referenceImageUrl;
         await supabase.from('shots').update({ reference_image_order: order }).eq('id', shotId);
     }
 }

 // Clean up stale continuity reference if match replaced a ref link
 const column = targetFrameType === 'start' ? 'reference_image_order' : 'end_frame_reference_image_order';
